# 绿灯激活诊断器

**绿灯世界书条目写了一堆，到底触发了没有，怎么触发的？** 
本插件解决此问题。

## 它解决什么问题？

用过世界书的人多少会遇到这些情况：

- 不清楚在哪里可以查看完整提示词，以确定绿灯条目是否触发
- 知道怎么查看提示词，然而每次都从完整提示词里找对应条目过于麻烦，尤其是绿灯条目众多时
- 不理解为什么绿灯条目为什么触发了/没有触发

绿灯激活诊断器会在每条 AI 回复的`更多`按钮中新增一个按钮，点开就能看到：

- ✅ **哪些条目触发并加入到提示词中**（触发词是什么、从哪条消息触发的）
- ❌ **哪些条目本来触发了，但被规则拦截**（比如概率没命中、同组里被别的条目挤掉、还在冷却中……）

对于世界书复杂的角色卡作者来说非常方便。

## 安装

在 SillyTavern 的「扩展」→「安装扩展」中，填入这个地址：

```
https://github.com/wilderye/green-light-activation-diagnostics
```

装完刷新页面就能用。

## 怎么用

打开「扩展」→「绿灯激活诊断器」，可以看到开关和相关调试选项。

日常使用：
1. 正常和 AI 对话
2. 在任意一条 **AI 回复**的`更多`菜单里，会多出一个 🚦 按钮
3. 点它，弹出诊断面板——上面列出了这条回复生成时，相关的绿灯条目激活情况

面板已经做了手机端适配。

## 说明

- **数据不外传**：诊断记录只存在你自己浏览器里，不写入聊天存档，不上传任何地方。换浏览器或清缓存不保留诊断记录，以达到自动清理效果。
- **自动清理**：超过 7 天或累计超过 500 条的记录会自动删掉。你也可以在设置里手动清除。
- **结果可信度**：「哪些条目被送给了 AI」这部分是 100% 准确的，来自 SillyTavern 自己的事件。「为什么没被送」的分析是插件自己推理的，绝大多数情况准确，但个别复杂规则可能分析不到。
